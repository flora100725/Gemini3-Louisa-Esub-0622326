# Agentic 510(k) 醫療器材查驗登記與 Meta-Skill 優化平台：使用者指南

歡迎使用 **Agentic 510(k) Pre-Market Notification Regulatory Review & Meta-Skill Optimization Platform**。本平台專為醫療器材法規事務（RA）、品保工程師（QA）及審查人員設計，結合了先進的多代理人（Multi-Agent）AI 架構，能自動化拆解、比對醫材查驗登記文件（如 510(k) 檔案、TFDA 查驗登記送審資料），並具備自我優化「審查技能指引（`skill.md`）」的 Meta-AI 能力。

本指南將分為**操作說明**與**新手本地端架設教學**兩大部分，協助您快速上手。

---

## 1. 系統核心四大功能區塊操作說明

本平台網頁介面基於 Streamlit 打造，主要分為四大操作核心：

### ⚙️ 區塊一：模型與輸入配置面板（Sidebar Panel）

* **模型選擇器**：預設使用最新的 **gemini-3.1-flash-lite**（具備超高速度與成本優勢）。如遇複雜技術報告（如大型軟體確效、複雜臨床統計），可切換至高階推理模型如 `gemini-1.5-pro` 或 `gpt-4o`。
* **審查技能（`skill.md`）配置**：您可以選擇使用系統內建的預設 TFDA 審查指引，或是上傳專屬於您公司特定產品線的自訂 `skill.md`。

### 🧠 區塊二：Meta-Skill 優化與動態執行模組（新功能）

* **步驟 1：上傳或貼上 Skill**：在「Meta-Skill 工作區」貼上您目前的 `skill.md` 文本。
* **步驟 2：AI 自動優化**：點擊「優化技能指引」，AI 代理人將自動分析條款是否模糊、是否有法規遺漏，並在畫面上顯示 **Side-by-Side（左右對比）的彩色 Diff 差異編輯器**（綠色為新增補強、紅色為刪除修正）。
* **步驟 3：動態編譯與測試**：您可以在編輯器中直接修改文字，確認無誤後點擊「編譯並註冊技能」。隨後即可上傳一份「測試樣品檔案」，即時觀看 AI 執行新技能的精準度。

### 📊 區塊三：互動式審查工作台與 6 大 WOW 視覺看板

當您正式上傳 510(k) 完整送審資料（支援 `.txt`, `.md`, `.pdf`, `.json`）後，系統將啟動多代理人協同審查，並輸出包含 6 大圖表的儀表板：

1. **合規缺失輻射熱圖（Radial Heatmap）**：一眼看清 8 大章節中哪些條款通過（綠）、缺失（紅）或待審（黃）。
2. **語意分歧與實體衝突網絡圖（Network Graph）**：自動抓出製造廠地址、名稱在各文件間不一致的隱藏衝突（如 Suite 401 與 4F 差異）。
3. **SaMD 數據污染與臨床世代流失圖（Venn/Euler Diagram）**：視覺化呈現機器學習訓練集與測試集是否有資料洩漏，以及臨床受試者人數是否異常失蹤。
4. **即時日誌終端機（Live Log Trace）**：滾動式顯示目前哪一個 AI 代理人正在處理哪一頁的哪一條法規，透明可追溯。

### 💾 區塊四：互動狀態修改與多格式匯出系統

* **手動覆核（Override）**：AI 生成報告後，人類審查員可直接點擊「Pass」、「Fail」或「N/A」按鈕進行修正，並可在輸入框內撰寫人工審查筆記。
* **一鍵導出**：右側的「即時補件缺失彙整區」會根據判定為「Fail」的項目自動生成標準格式的**補件通知書草稿**。支援下載為 Markdown 檔案、獨立離線網頁（HTML SPA），或直接同步建立至 **Google Docs** 雲端文件。

## 2. 新手本地端（Local Machine）架設步驟教學

本教學專為完全沒有程式背景的初學者設計，請按照以下步驟逐步執行，即可在您自己的電腦（Windows 或 Mac）上運行此平台。

### 步驟一：安裝基礎環境工具

1. **安裝 Git**：
* [Windows 用戶]：前往 [Git 官網](https://git-scm.com/) 下載並安裝，安裝過程中一路點擊「Next（下一步）」即可。
* [Mac 用戶]：打開「終端機 (Terminal)」，輸入 `git --version`，系統會自動彈出提示引導您安裝開發者工具。


2. **安裝 Python (3.10 或 3.11 版本)**：
* 前往 [Python 官網](https://www.python.org/downloads/) 下載對應您作業系統的安裝檔。
* ⚠️ **超級重要（Windows 用戶）**：在安裝視窗啟動時，**務必勾選最下方的 "Add Python to PATH"（將 Python 加入環境變數）**，然後再點擊 Install Now。



### 步驟二：取得專案原始碼

1. 打開您電腦的「命令提示字元 (cmd)」(Windows) 或「終端機 (Terminal)」(Mac)。
2. 輸入以下指令並按下 Enter，將專案下載到您的電腦：
```bash
git clone https://github.com/your-username/medical-device-review-agent.git

```


*(請將上述網址替換為您在 Hugging Face Spaces 或 GitHub 上的實際專案網址)*
3. 切換進入該專案資料夾：
```bash
cd medical-device-review-agent

```



### 步驟三：建立虛擬環境與安裝依賴套件

為避免這套軟體的套件與您電腦過往的其他軟體衝突，我們需要建立一個乾淨的虛擬空間：

1. **建立虛擬環境**：
```bash
python -m venv venv

```


2. **啟動虛擬環境**：
* [Windows 用戶]：輸入 `venv\Scripts\activate`
* [Mac / Linux 用戶]：輸入 `source venv/bin/activate`
* *(啟動成功後，您的指令輸入行最前面會出現 `(venv)` 字樣)*


3. **升級基礎工具並安裝所有需要的 AI 套件**：
```bash
pip install --upgrade pip

```


```bash
pip install -r requirements.txt

```



### 步驟四：配置 API 金鑰鑰匙圈（Environment Variables）

本平台需要呼叫 Google Gemini 與 OpenAI 的大型語言模型。

1. 在專案根目錄下，建立一個名為 `.env` 的新純文字檔案。
2. 使用記事本或任何文字編輯器打開它，寫入您的 API 金鑰（請替換為您個人的實際金鑰）：
```env
GEMINI_API_KEY=AIzaSyYourActualGeminiKeyHere
OPENAI_API_KEY=sk-proj-YourActualOpenAIKeyHere

```


3. 儲存並關閉檔案。⚠️ *請勿將此檔案上傳至任何公開 GitHub 倉庫，以免金鑰外洩。*

### 步驟五：啟動 Streamlit 平台

1. 在依然處於 `(venv)` 狀態的終端機中，輸入以下指令：
```bash
streamlit run app.py

```


2. 系統會自動進行本地端伺服器編譯。數秒後，您的預設網頁瀏覽器會自動彈出一個新分頁，網址通常顯示為 `http://localhost:8501`。
3. **大功告成！** 您現在可以在自己的電腦上安全地進行 510(k) 與 TFDA 法規審查與技能優化。

## 3. 20 個深度後續探討問題（Comprehensive Follow-up Questions）

為了優化您與本平台的互動並推動後續系統調整，請思考或回覆以下問題：

1. 您目前最常處理的醫療器材屬於哪一個風險等級（Class I, II, 還是 III）？這會影響預設審查機制的嚴格度。
2. 您的送審 PDF 檔案是否包含大量掃描版（掃描圖像）？我們是否需要為您加強地端 OCR（光學字元辨識）的預處理效能？
3. 在使用 Meta-Skill 優化模組時，您是否希望系統能與 TFDA 衛福部食藥署官網的最新法規公告進行即時聯網比對更新？
4. 當系統自動編譯出新的 `skill.md` 修正檔時，您需要系統為您保留歷史版本紀錄（如 Git Version Control）以便隨時回滾嗎？
5. 針對 6 大 WOW 看板中的「地址實體衝突網路圖」，您希望設定多寬鬆的字串容許度（例如判定 "Co., Ltd." 與 "Company Limited" 是否算衝突）？
6. 您的 510(k) 摘要或送審文件中，是否包含敏感的病患個資（PII）需要我們在前端設計徹底的本地去識別化過濾器？
7. 本地端運行速度是否符合您的預期？我們是否需要針對您的硬體配置（如是否具備 GPU 顯示卡加速）優化快取機制？
8. 對於匯出至 Google Docs 的功能，您是否需要公司專屬的官方公文排版樣式（Template Template）與標題層級對齊？
9. 針對臨床試驗世代流失審查（AI Feature 7），您常遇到的受試者表格數據通常是以 PDF 還是 Excel/JSON 格式呈現？
10. 當 AI 代理人判定某一條款為 "Fail" 時，除了預設的說明外，是否需要自動帶出過往類似產品通過審查的補件範例（RSI Reference）？
11. 您的團隊是否有多人共同審查同一案子的需求？這會涉及到我們是否該從本地單機版升級至多帳號雲端協作版。
12. 平台預設的 8 大審查章節是否完整涵蓋了您目前的所有品項？是否有特定硬體或特定耗材的技術文件需要加入預設指引？
13. 對於軟體醫療器材（SaMD）的資安審查（SBOM），您是否希望系統能自動連線至漏洞資料庫（如 CVE）比對組件安全性？
14. 在本地端安裝步驟中，您是否有遇到任何關於 Python 環境變數（PATH）或是套件衝突的錯誤訊息？
15. 系統產出的 SPA 網頁報告（HTML），您希望它能嵌入審查員的數位電子簽章（Electronic Signature）以符合 21 CFR Part 11 合規性嗎？
16. 對於跨國文件對齊（AI Feature 9），您是否需要擴充繁體中文、英文以外的語言對（例如日文原廠說明書或德文 QMS 證書）？
17. 您希望平台如何處理生物相容性報告（ISO 10993）中的複雜化學物質浸潤與溶出試驗數據圖表？
18. 當 API 呼叫遇到流量限制（Rate Limit）時，目前的 500 毫秒退避切換機制是否能滿足您連續大量上傳檔案的狀況？
19. 您是否希望在儀表板中加入一個「預估審查通過率與 RSI 風險指數（0-100%）」的直觀動態儀表？
20. 為了協助新進 RA 快速上手，您是否需要我們在介面中加入一個內建的「引導式教學導覽（Interactive App Tour）」功能？
